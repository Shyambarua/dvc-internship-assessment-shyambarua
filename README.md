## Question 1: Responsive Product Card Component

### Approach
- Built using semantic HTML5, CSS, and Vanilla JavaScript.
- Implemented mobile-first responsive design.
- Added interactive quantity selector and Add to Cart functionality.

### Features
- Responsive layout for mobile, tablet, and desktop
- Discount pricing display
- Quantity selector (1–10 limit)
- Disabled buttons at min/max values
- Success message on Add to Cart
- Console logging of cart details

### Assumptions
- Static product data used for demonstration
- Console logging simulates cart functionality

### Time Spent
Approx: 2–3 hours

## Question 2: WordPress Testimonials Plugin

### Approach
- Created a custom post type named **Testimonials**
- Added custom meta fields for client details
- Implemented shortcode for frontend display
- Followed WordPress coding standards and security practices

### Features
- Admin panel to manage testimonials
- Client photo, name, company, and rating display
- Responsive testimonial layout
- Shortcode parameters for sorting and limiting results
- Gutenberg compatibility

### Assumptions
- Tested on a standard WordPress installation
- Basic styling included

### Time Spent
Approx: 3–4 hours

## Question 3: Weather Dashboard

### Approach
- Built using Vanilla JavaScript and OpenWeatherMap API
- Used async/await for API calls
- Implemented loading states, error handling, and localStorage

### Features
- Search weather by city
- Current weather display
- 5-day forecast
- Loading spinner
- Error handling
- Saves last searched city
- Responsive UI

### Assumptions
- API key required from OpenWeatherMap
- Free API tier limitations apply

### Time Spent
Approx: 3–4 hours
